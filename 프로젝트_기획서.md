## 📋 프로젝트 개요

### 컨셉
- **2000년대 초반 감성**의 동창 찾기 플랫폼
- 다모임(Damoim) 스타일의 복고풍 디자인
- 취미 프로젝트 (사업성 고려 X)

### 개발 순서
1. **Phase 1**: 웹 버전 개발 (우선)
2. **Phase 2**: 모바일 앱 개발 (동일 기능)
   - 플레이스토어 등록으로 검색 노출

### 아키텍처
```
프론트엔드(웹) ──┐
                 ├──→ 백엔드 API (공통)
프론트엔드(모바일)─┘
```

## 🎯 핵심 기능

### 1. 회원 시스템

**회원가입 필수 정보:**
- 이메일 (로그인용, 비공개)
- 비밀번호
- 닉네임 (공개, 다른 사람에게 표시)
- 학교명
- 졸업년도

**회원가입 선택 정보:**
- 프로필 사진
- 자기소개
- 학년/반 정보

**개인정보 보호 원칙:**
- ⚠️ **실명은 아예 수집하지 않음**
- 닉네임에 실명을 쓰는 것은 사용자 선택
- 안내 문구: "닉네임은 다른 사람에게 보여집니다. 실명 사용 시 개인정보가 공개될 수 있습니다."
- 전화번호 수집 안 함
- 최소한의 정보만 수집

### 2. 동창 찾기 (핵심 기능)

- 학교명으로 검색
- 졸업년도로 필터링
- 학년/반 정보로 검색
- 검색 결과 목록 표시
- 프로필 보기

### 3. 커뮤니케이션

#### 3-1. 1:1 채팅
- 실시간 대화창
- 읽음 표시
- 채팅 목록 (최근 대화 순)

#### 3-2. 쪽지 (선택적)
- 비동기 메시지
- 1:1 채팅과 별개로 운영 가능
- *(최종 결정 필요: 1:1 채팅으로 통합할지 고민)*

#### 3-3. 방명록
- 프로필 페이지에 공개 메시지 남기기
- 다른 사람들도 볼 수 있음

### 4. 그룹 채팅 (오픈채팅 방식)

**컨셉: 카카오톡 오픈채팅 스타일**

**채팅방 생성:**
- 방 제목 (예: "서울고 2010학번 수다방")
- 학교 + 졸업년도 선택 (필수)
- 방 설명
- 최대 인원 설정 (선택)
- 공개/비공개 (링크로만 입장 가능)

**채팅방 찾기:**
- 내 학교/졸업년도 기준으로 필터링
- 검색으로 방 찾기
- 참여 인원 수 표시
- 입장 방식: 바로 입장 or 승인 필요

**채팅방 기능:**
- 방장 권한 (강퇴, 공지)
- 나가기/신고하기

**핵심 차별점:**
- ✅ **학교/졸업년도 인증된 사람만 입장 가능**
- "우리 학교 우리 학번" 보장

**닉네임 vs 실명:**
- *(최종 결정 필요: 채팅방에서 닉네임 사용할지, 실명 권장할지)*

### 5. 커뮤니티

- 학교별 게시판
- 졸업년도별 게시판
- 동문회 공지사항

## 🎨 디자인 컨셉 (2000년대 감성)

### UI/UX 스타일
- 투박하고 심플한 UI (세련되지 않은 느낌)
- 게시판 중심의 레이아웃
- 2000년대 초반 한국 포털 사이트 감성
- 그라데이션 배경 (당시 스타일)
- 투박한 버튼 디자인

### 참고 요소
- 싸이월드 미니홈피 느낌의 프로필 페이지
- "쪽지함", "방명록" 같은 옛날 용어 사용
- 느린 템포, 여유로운 사용자 경험

### 모바일에서의 적용
- 복고 감성을 살리되, 사용성은 해치지 않도록
- 핵심 UX는 모던하게 유지
- 비주얼/용어로 감성 전달

## 🎓 다모임(Damoim) 참고 정보

### 역사
- **설립**: 1999년 10월 25일
- **서비스 시작**: 2000년 3월
- **서비스 종료**: 2009년 4월 22일

### 특징
- 아이러브스쿨이 졸업생 중심이었다면
- **다모임은 재학생 중심** 커뮤니티
- 초/중/고등학교 동창 찾기

### 진화 과정
- 2002년: 싸이월드 제휴, I.Style (미니홈피) 서비스
- 2006년: 동영상 서비스 추가 (MnCast, Aura)
- 2007년: iPlay.com으로 이름 변경

### 우리 프로젝트에 적용
- ✅ 학교/학년 기반 검색 구조
- ✅ 게시판 중심 커뮤니케이션
- ✅ 프로필 페이지 (미니홈피 느낌)
- ❌ 동영상 기능 (제외)
- ❌ 미니홈피 (제외)
- → **기본에 충실**

## ⚖️ 법적 고려사항

### 개인정보보호법 대응
1. **실명 수집 안 함** (핵심 전략)
2. 닉네임 자유 입력 (실명 사용은 사용자 선택)
3. 최소한의 정보만 수집
4. 이용약관 및 개인정보 처리방침 작성
5. 사용자 동의 절차

### 안내 문구 예시
```
닉네임 입력 시:
"다른 사람에게 보여질 이름입니다.
실명 사용 시 개인정보가 공개될 수 있습니다."
```
